Build a major product upgrade for my existing app, Sift Chat.

Current product:
- Group-based planning app with shareable links
- No account required to join, users type a name
- Real-time group chat polling every 3 seconds
- AI reads the chat and extracts a structured main plan with “When” and “Where”
- AI surfaces alternatives and supports voting
- Most popular option becomes the main plan
- Attendance is inferred from support and votes
- Returning users can continue as themselves via saved browser ID
- Share Plan button copies a clean summary
- Frontend uses Shadcn/ui, Tailwind, Framer Motion, violet/indigo aesthetic
- Backend is Express 5 + PostgreSQL + Drizzle ORM + OpenAI via Replit AI integration

I want to evolve Sift Chat into a travel-focused group decision product, but NOT a full itinerary builder like Wanderlog. The product should feel different: it should specialize in helping groups go from vague interest and chaotic discussion to an actual locked trip decision.

Core product direction:
Sift Chat should own the “decision before booking” phase of group travel.
It should help groups answer:
- Where should we go?
- When should we go?
- Who is actually in?
- What budget works?
- What trip style are we doing?
- Which option has the strongest real support?

Do NOT build:
- reservation imports
- full itinerary builder
- flight booking
- hotel booking
- expense splitting
- complex day-by-day planning UI

Instead, build:
1. A travel-aware planning workspace layered on top of the existing group chat
2. AI-generated trip options based on group discussion
3. Voting and support tracking for full trip bundles
4. A stronger plan state that reflects confidence and commitment
5. A helper AI inside the chat that can guide the group toward consensus

Main concept:
The app should no longer only extract “When” and “Where.”
It should build a structured “Trip Plan” object with:
- destination
- date range
- budget band
- trip vibe / type
- lodging preference
- likely attendees
- committed attendees
- unresolved questions
- ranked alternatives

Product behavior:
As group members chat naturally, the AI should detect travel intent and gradually structure the trip.
People should still be able to type casually, but the app should extract and maintain trip state live.

Examples of messages users might send:
- “beach trip in may?”
- “I can only do memorial weekend”
- “under 600 total please”
- “I’m down for miami or san diego”
- “I’d rather do a cabin than a hotel”
- “I can only go if we leave friday night”
- “I’m interested but not 100% booked yet”

The app should interpret this and update the structured trip planning UI without requiring forms.

AI HELPER REQUIREMENTS

Add a cute one-word AI helper bot named: Pip

Pip should appear in the chat as an AI planning assistant for the group.
Pip must not dominate the conversation or feel spammy.
Pip should speak only when useful.

Pip’s role:
- summarize the state of discussion
- suggest 2–3 strongest trip options
- identify conflicts
- ask clarifying questions when necessary
- encourage the group toward consensus
- explain why the current main option is winning
- announce when the main plan shifts
- point out when support is soft vs actually committed

Examples of good Pip behaviors:
- “Right now Miami on Memorial weekend has the strongest support, but the budget concern is still unresolved.”
- “Two strong options are emerging: San Diego in July and an Austin cabin in June.”
- “I’m seeing 5 people interested in a beach trip, but only 3 who clearly supported the May dates.”
- “Looks like cabin > hotel is the dominant lodging preference so far.”
- “This plan is leading because it has the most destination support and the strongest attendance signal.”

Pip should intervene when:
- a new strong option appears
- there are multiple conflicting options
- the group seems stuck
- enough evidence exists to update the main plan
- a clarification question would reduce ambiguity

Pip should not intervene:
- after every message
- when nothing materially changed
- with overly long responses
